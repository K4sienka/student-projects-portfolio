# Database Systems and Data Analysis - Semester Projects

This repository contains laboratory reports and notebooks completed during the semester. The projects focus on SQL, database performance, analytical databases, document databases, Python data processing, and spatial data analysis.

## Repository Contents

### Report 1 - SQL Aggregation and Window Functions

Introduction to aggregate and window functions using the Northwind database. The report compares `GROUP BY`, subqueries, joins, and window functions.

### Report 2 - Advanced SQL Window Functions

Practical use of `ROW_NUMBER`, `RANK`, `DENSE_RANK`, `LAG`, `LEAD`, `FIRST_VALUE`, and running totals. The solutions were tested and compared in PostgreSQL, Microsoft SQL Server, and SQLite.

### Report 3 - SQL Server Indexes and Query Optimization

Analysis of SQL Server execution plans and query performance. The exercises cover the Database Engine Tuning Advisor, clustered and non-clustered indexes, covering indexes, and index column order.

### Report 4 - Advanced Indexing and Columnstore

Further experiments with SQL Server indexing, logical reads, query costs, SARGable conditions, included columns, and columnstore indexes. The report compares query performance before and after index creation.

### Report 5 - PostgreSQL vs ClickHouse

Event data analysis in PostgreSQL and ClickHouse. The report includes data profiling, sales KPIs, user activity analysis, aggregations, and query performance benchmarks.

### Report 6 - Advanced Analytics in ClickHouse

Exercises covering conversion funnels, window functions, revenue trends, RFM customer segmentation, and performance comparisons between ClickHouse and PostgreSQL.

### Report 7 - Couchbase Document Database

Introduction to Couchbase and document-oriented data modelling. The report covers JSON documents, primary and secondary indexes, SQL++ queries, `JOIN`, `UNNEST`, nested documents, and execution plan analysis.

### Report 8 - Python and Databases

A Jupyter Notebook comparing database-side processing with pandas. It covers SQL pushdown, chunk-based processing, memory optimisation, SQL joins versus pandas `merge()`, pivot tables, binning, data cleaning, and exporting prepared datasets.

### Report 9 - Oracle Spatial

Introduction to spatial data processing with Oracle Spatial. The materials include spatial SQL queries, `SDO_GEOMETRY`, geographic data visualisation, Oracle SQL Developer, and Python/Jupyter integration.
